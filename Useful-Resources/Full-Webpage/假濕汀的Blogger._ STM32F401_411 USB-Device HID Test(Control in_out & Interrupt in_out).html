STM32F401/411 USB-Device HID Test(Control in/out & Interrupt in/out)

1. This article is based on STM32F401/411 HID Test (Keyboard and Mouse and HidRaw) this article continues to explore the testing methods of HID bidirectional communication.
2. Sample program download link: (using IIR EWARM v7.7) stm32f401_411_usb_device_hid_test_2016




Since the previous example had only this part of the mcu to the pc (interrupt in), we first had to fill in this part of the pc to the mcu (interrupt out).

  • First come usbd_hid_core.c to add the relevant fusion.

  • Declare USBD_HID_DataOut function.

  • Add USBD_HID_DataOut callback function.
  • Change Endpoints from 1 to 2 as originally defined.

  • Added a description of endpoint out.


  • Since the description of endpoint out has been added, it is necessary to add the description of endpoint out.

  • The current program defines the data length of pc to mcu as 8bytes.


  • After burning the program in, we can open bus hound for testing, when the program is successfully executed and linked to pc, you should see the following screen on bus hound, at which point we select the top-level USB input device, and double click it.


  • At this point we select the interrupt out endpoint inside endpont 1, then modify Data Length to 8, and enter the same data as the screen in the data area below, and follow the steps 1 - 2 - 3 - 4.

  • At this point should have seen LED3 has begun to light up.

  • As we can see from the USBD_HID_DataOut, after receiving the 8bytes from the pc, the program begins to judge which LED to perform on/off function, where we currently only use Report_buf s 0 s and Report_buf s1 s, respectively, representing the following meaning:
  Report_buf[0]: LED number
Report_buf[1]: On/Off
Report_buf[2]~Report_buf[7]: Reserve


  • The bus hound operation diagram of interrupt in/out.
    • When the user button is pressed, it starts sending data to the pc via the endpoint of interrupt in, and then the pc turns LED3 on/off via interrupt out.


============================= control in/out =============================
4. In addition to using EP-1 interrupt in/out, you can also use EP-0 control in/out to communicate with pc, so first we have to make up the program ofcontrol in/out.
  • Declare the variables and programs that EP0 needs to use first .
  • Add the EP0 program to call back function.
  • Then in USBD_HID_Setup function, add set report and get report these two commands.

  • Then complete EP0_RxReady and EP0_TxSent two fusions.

  • control out(set report command) flow:
    • USBD_HID_Setup function => case HID_REQ_SET_REPORT => USBD_HID_EP0_RxReady function
  • control in(get report command) flow:
    • USBD_HID_Setup function => case HID_REQ_GET_REPORT => USBD_HID_EP0_TxSent function
  • The pc to mcu test is the same as the interrupt out, after burning the program in, after opening the device with bus hound, on the EP0 Control double click, and then entering the instruction (set report), you can turn the LED on/off.
  • The mcu to pc test method is shown below, and the default data can be read by pressing the line step.

  • Control in/out bus hound operation diagram.
    • Use set report command to have the pc send data to mcu to turn the LED on/off, and then the pc reads the data through get report command.
